# Как работать с Consta

Дизайн-система — это библиотека компонентов, с ней можно работать
как с любой библиотекой: подключить и использовать готовые компоненты,
создавать новые, использовать только макеты, только код или просто
посмотреть и сделать так же&nbsp;— зависит от вашей задачи.

Здесь описан вариант использования дизайн-системы, который кажется
нам оптимальным, следовать ему не обязательно.

## 1. Дизайн-макет

1. Дизайнер подключает к проекту в Figma библиотеки дизайн-системы. Например, [библиотеку компонентов Consta UI Kit](https://www.figma.com/community/file/853774806786762374) и [библиотеку графиков Consta Charts](https://www.figma.com/community/file/982611119114314434).

2. Создаёт макеты интерфейсов с помощью готовых компонентов из Consta.

3. Разрабатывает свои компоненты по принципам дизайн-системы (если готовых компонентов не хватает).

4. Описывает макет для разработчиков: как ведут себя элементы,
   какие у них могут быть модификации, варианты цветовых схем, анимацию —
   всё, что может понадобиться при реализации проекта.

После этого макеты отправляются в разработку.

> Если по каким-то причинам вам не подходят библиотеки в Figma,
> можно использовать дизайн-систему как брендбук: посмотреть и сделать так же.
> Размеры, параметры цветов и так далее можно взять из Figma.

## 2. Разработка

1. Разработчик устанавливает NPM-пакеты с библиотеками дизайн-системы. Например, `@consta/uikit` и `@consta/charts`.

2. Подключает стандартную тему со стандартной цветовой схемой (или создаёт свою
   и всё равно её включает). См. [Быстрый старт](/?path=/docs/common-start--page)

3. Создаёт интерфейс по макетам и спецификации, которые передал дизайнер.
   При этом использует компоненты из библиотек Consta, их работа описана в этой
   документации. [Посмотреть список компонентов](?path=/docs/common-preview--page)

Если каких-то компонентов не хватает, их можно создать самостоятельно.

> Дизайн-система сделана на React.js. Если вы работаете с другими технологиями,
> библиотеки использовать не получится. В этом случае
> можно сделать дизайн на основе библиотек в Figma и разработать
> все нужные компоненты самим.

## 3. Тестирование и следующие этапы

Дальше проект идёт как обычно, без участия дизайн-системы:
скорее всего, отправляется тестировщикам,
на согласование с заказчиками и так далее.
